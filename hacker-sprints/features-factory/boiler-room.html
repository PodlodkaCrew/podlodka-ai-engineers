Boiler Room

github.com/podlodka-ai-club/boiler-room

Маленький Python-CLI: «бригадир» забирает задачи из колонки Todo GitHub Project board, разводит каждую в свой git-worktree, скармливает локальному кодинг-агенту (Claude / Copilot / Codex) и открывает PR. Никакого собственного состояния — вся правда живёт на доске.

Главная идея

Никакого собственного «мозга» с памятью у boiler-room нет

Вся правда о том, кто что делает, лежит снаружи — на GitHub Project board: статус задачи (Todo / In Progress / Done) и пара лейблов (agent-run, failed). Локальная программа — просто «бригадир»: читает доску, выписывает наряды, заводит каждого работника в свою комнату и снова смотрит на доску. Любой crash или kill — следующий запуск увидит то же состояние и продолжит.

Высокоуровневая схема

Как устроена «котельная»

Один main-thread «клеймит» задачи, ThreadPoolExecutor запускает агентов в параллель. Между ними — single-source-of-truth GitHub Project board.

Что важно понять

  • → State снаружи. Никакого SQLite, никакого state-файла. Если упал процесс — следующий запуск увидит доску ровно в том же виде.
  • → Один поток забирает работу, много делают. Главный thread «клеймит» задачи в In Progress последовательно, до отправки в пул. Между worker'ами одного процесса гонок нет.
  • → Subprocess-only, никаких SDK. Единственная runtime-зависимость — pydantic. Всё наружу — это subprocess.run(["gh", ...]) или git.
  • → Один ref — для всего. Имя ветки feature/<ref>, путь .agent-runs/<ref>/, output-файл — производятся от одного property Task.ref.
Поддерживаемые агенты

Три адаптера к агентам

Базовый AgentAdapter знает только две вещи: как собрать общий промпт по задаче и где должен лежать build_command. Каждый из трёх адаптеров — буквально по шесть строк. Объединяет их одно: все запускаются с самыми «разрешающими» флагами.

Three-tier protocol

Три уровня fallback'а

Pipeline разделён на три отчётливые стадии, и у каждой свой план Б. Failure-path сам никогда не падает — все его операции в индивидуальных try/except.

Из чего собран

Девять строительных блоков

От «доски-источника правды» до «накладной от агента» — каждый компонент решает одну конкретную задачу. Используйте фильтр для просмотра по категориям.
